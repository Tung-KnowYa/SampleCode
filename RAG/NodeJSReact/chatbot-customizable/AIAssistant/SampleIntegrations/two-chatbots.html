Two assistants on one host page

You get two separate launchers — one per custom element. Point api-base at your API origin if the backend is not same-origin.

<ai-chatbot-widget theme="light">

Full assistant: knowledge + data tabs, tools, reports.

  • Instant first message: set initial-prompt="…" to send that text once when the panel opens, or call element.open('…') / element.open({ prompt: '…' }) (takes priority for that open).

<knowledge-chatbot-widget>

Knowledge-only flow (source: 'knowledge'). This sample sets api-base="http://localhost:3002" so the widget calls that origin’s /api/chat while you open the file from disk or another port.

  • Adjust api-base or defaults in frontend/src/components/chat-widget/defaultApiBases.js.
  • Ensure CORS on the API allows your host origin when they differ.
  • Same initial-prompt and open(prompt) behavior as the full widget.
